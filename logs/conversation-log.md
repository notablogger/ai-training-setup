## Observations

- **AI never asked for clarification when it could act** — it made reasonable assumptions and explained them after. Prompts 1 and 5 were both one sentence and produced substantial, correct output.
- **Every architectural decision was the developer's** — CQRS reads from MongoDB (prompt 5), the multi-language roadmap (prompt 8), cutting test scope when it blocked progress and returning to it later (prompts 12). AI executed all of them without pushback.
- **The audit prompt (9) was the highest-value single prompt in the project** — one question, 6 real bugs fixed. None of those bugs would have appeared in basic happy-path testing; all would have caused failures in production.
- **Prompt 10 (explain your changes) was as valuable as prompt 9** — understanding *why* a fix works makes the pattern stick. Asking AI to explain after it acts is underused.
- **When AI hit a wall (prompt 7, git remote)** — it didn't guess or fabricate a URL. It stopped and asked. Knowing where AI draws that line is useful.
- **The Testcontainers setup (prompt 12) required iteration** — the MongoDB URI resolution inside the dynamic property source failed on the first two attempts before a working approach was found. AI diagnosed iteratively rather than in one shot. The developer cut scope once rather than letting it spiral.
- **Documentation was generated from the actual code state** — all README content, AI docs, and reference docs were written after the code was built and audited, not speculatively. They reflect what actually exists.
